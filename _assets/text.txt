                            <p><span style="color: yellow;font-size: 35px;">Preliminary Round:</span><br>
For the initial round, a technical term related with
computer science shall be partitioned into a number of
words. Each word shall have an illustrative clue, through
which the participants must identify through linking them
to the particular concept. Using this, the participants are
expected to identify what link each word has in order to
find the clue.<br><br>

                                <span style="color: yellow;font-size: 35px;">Finals:</span><br> Description Each participant will be given a program with minimum
errors. The challenge of the participants is to debug and
type the code in the TEXT EDITOR and MONITOR OFF.
Program with complex errors will be provided on a paper.
Twenty minutes will be given to complete the program.
Based on the results of compilation and execution the
WINNERS will be awarded.<br><br>

                                <span style="color: yellow;font-size: 35px;">Rules:</span><br>

Each team should strictly have only two members
PRELIMS: No electronics allowed. 2 per team. 120 seconds
per question for the first round
FINALS: First preference goes to the participants whose
code gets executed.
Second preference goes to the participants whose code
has less number of errors.
IN CASE OF TIE:
A Tie breaking problem will be given and timing will
be considered.<br><br>

<span style="color: yellow;font-size: 35px;">Event Domain:
(Knowledge area of
    your participants)</span><br>

Programming and Logical reasoning</p><br><br>
                        <p style="color: aqua;font-size: 45px;"><strong>BREAK THE ENIGMA</strong></p>
                        <img src="_pics/break.webp" class="responsive" alt="">
                            <p><span style="color: yellow;font-size: 35px;">Preliminary Round:</span><br>
Questions based on all the fields on computers will be
asked, these questions must be solved in a particular order
to find the “ENIGMA” (KEY) to the next round. The key will
be easy to decrypt at first and then scales appropriately.
Teams which find the maximum length of the key will move
to next round. Questions are based on Computer
Networks, Programming Logic, Operating System, Database
Management System and OOPs will be asked.<br><br>

                                <span style="color: yellow;font-size: 35px;">Finals Description:</span><br> Teams who reach the final round will be given 5 URLs that
are encrypted. The Algorithm used to them encrypt will be
given for each question. The teams must decrypt the URL
(Internet Usage is allowed to perform decryption).The URLs
will lead to a webpage where a programming question has
to be solved. The easiest URL to decrypt will have the
toughest program and the toughest URL will have the
easiest program.
5 * 50 = 250 Marks in total
For each question if the key is decrypted 25 Marks
For each question if the program is solved 25 marks<br><br>

                                <span style="color: yellow;font-size: 35px;">Rules:</span><br>

 No internet Usage (Outside of provided
stations)
 Each team should strictly have only two
members
 No calculators allowed
 Same team for both Prelims and Finals
 Prelims 30 Questions
 Prelims time 30 Minutes
 Finals 5 questions
 Finals time 45 Minutes<br><br>

<span style="color: yellow;font-size: 35px;">Event Domain:
(Knowledge area of
    your participants)</span><br>

Programming, Logical Reasoning, Hardware Knowledge,
Encryption &amp; Decryption, Networks, Operating System,
Object Oriented Programming. The interest and will to do
self-learning.</p><br><br>
                    <p style="color: aqua;font-size: 45px;"><strong>CODE QUEST</strong></p>
                           <img src="_pics/code.webp" class="responsive" alt="">
                            <p><span style="color: yellow;font-size: 35px;">Preliminary Round</span><br>
1. A technical maze in the form of a table (each cell in the
table has a coding question that evaluates to true or false)
is given to each team. 
2. The cells that evaluate to true must be shaded. The team
must evaluate each cell, shade the cells accordingly to help
the programmer reach his PC. 
3. Marks are awarded based on the number of correct cells
shaded. The participants who have found the correct path
to the maze will be selected to next round.
4. Maximum duration of 30 minutes will be provided.<br><br>

                                <span style="color: yellow;font-size: 35px;">Finals Description:</span><br>1. Each team must code(C,C++,Java, python) and
display the output for the given problem
2. Following this, the team must obfuscate their code
and execute them.
3. The code will be distributed among other teams.
4. Each team finding the correct logic will be given
points.
5. Prizes will be decided based upon the point secured
by the team.
6. Duration: 1 hour (30mins- Coding &amp; Obfuscation,
30mins De-Obfuscation).<br><br>

                                <span style="color: yellow;font-size: 35px;">Rules:</span><br>

1. Each team should strictly have only two members.
2. Strictly NO Internet access.
3. Organizers decision is final.<br><br>

<span style="color: yellow;font-size: 35px;">Event Domain:
(Knowledge area of
    your participants)</span><br>

Programming logics, logical reasoning, basic algorithms
Basic programming knowledge</p><br><br>
